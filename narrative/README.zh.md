# 品牌叙事 — TALE

**每个频道都会继承的品牌故事 —— 溯源、搭建、落地、评估。**

[English](README.md) | 简体中文

> 这是 **[Aaron 营销技能库](../docs/README.zh.md)** 中的一个学科 —— 全库 120 个技能、七个学科、共享一套契约。要看整套体系、四层地图与安装步骤，请从[主 README](../docs/README.zh.md) 开始。

品牌叙事是 **L1 · 战略**层：一套品牌声音，被五个常态频道（SEO/GEO、社媒、邮件、付费、红人）继承，并由产品发布在关键时刻集中放大。十六个技能端到端跑通 **TALE** 循环 —— **T**race 溯源今天真实存在的故事，**A**rchitect 搭建叙事正典，**L**and 把它落地到每个频道与 pitch，再 **E**valuate 评估它是否真实、自洽、有共鸣。正典本身由 [`narrative-registry`](../protocol/narrative-registry/SKILL.md)（品牌正典 SSOT）持有；质量门是 [`narrative-quality-auditor`](evaluate/narrative-quality-auditor/SKILL.md)。

## 循环 — Trace → Architect → Land → Evaluate

- **Trace（溯源）** —— 找到诚实的起点：今天真实散落在自有阵地上的故事、品类里占主导的叙事与被点名的替代方案、受众已经相信的东西，以及每一条定位声明回溯到的实证依据。
- **Architect（搭建）** —— 搭建正典：核心战略叙事、信息体系（tagline / 支柱 / 证据点）、声音与词库规则，以及一个可复用的证据故事库。
- **Land（落地）** —— 把正典送进现实而不稀释：按频道逐层下沉、塑造成 pitch 形态，并把证据点打包成频道即用、贴合声明台账的素材。
- **Evaluate（评估）** —— 检验它是否立得住：TALE 门分别跑 truth / system / effectiveness 三个档，信息测试读取共鸣，两个监测器盯落地表现与偏离已批准正典的漂移。

|------|------|:----:|------|
| **Trace** | [narrative-baseline-mapper](trace/narrative-baseline-mapper/SKILL.md) | T | 捕捉当前真实的品牌故事在自有阵地上的样子 —— 任何重塑之前诚实的起点。 |
| **Trace** | [category-narrative-mapper](trace/category-narrative-mapper/SKILL.md) | T | 梳理品类里占主导的叙事与被点名的替代方案，让品牌能占据一个可防守、有差异的位置。 |
| **Trace** | [audience-belief-mapper](trace/audience-belief-mapper/SKILL.md) | T | 挖出目标受众已经相信、怀疑、在意的东西 —— 叙事必须撬动的那些信念。 |
| **Trace** | [positioning-truth-tracer](trace/positioning-truth-tracer/SKILL.md) | T | 把每一条定位声明回溯到实证，撤下任何缺乏支撑的说法（在 T1 真实性否决之上游）。 |
| **Architect** | [strategic-narrative-designer](architect/strategic-narrative-designer/SKILL.md) | A | 设计核心战略叙事 —— 品牌领衔的"改变世界"故事弧、赌注与收束。 |
| **Architect** | [message-system-architect](architect/message-system-architect/SKILL.md) | A | 搭建信息体系 —— tagline、支柱、证据点与分受众角度，形成一个自洽结构。 |
| **Architect** | [brand-language-codifier](architect/brand-language-codifier/SKILL.md) | A | 固化声音、语气、词库与该说/不该说的语言，让每个频道听起来是同一个品牌。 |
| **Architect** | [story-bank-builder](architect/story-bank-builder/SKILL.md) | A | 建一个可复用的证据故事库 —— 各频道可随时取用的证明故事、客户叙事与类比。 |
| **Land** | [narrative-cascade-planner](land/narrative-cascade-planner/SKILL.md) | L | 规划叙事如何逐层下沉到每个频道与时刻，不稀释、不漂移。 |
| **Land** | [pitch-narrative-builder](land/pitch-narrative-builder/SKILL.md) | L | 把叙事塑造成 pitch 形态 —— deck 主线、demo 故事、投资人/媒体框架。 |
| **Land** | [narrative-enablement-kit](land/narrative-enablement-kit/SKILL.md) | L | 赋能包，让每个团队都能一致地讲这个故事 —— 话术、FAQ、信息地图。 |
| **Land** | [proof-point-packager](land/proof-point-packager/SKILL.md) | L | 把证据点打包成频道即用、贴合声明台账的素材。 |
| **Evaluate** | ⛩ [narrative-quality-auditor](evaluate/narrative-quality-auditor/SKILL.md) | T·A·L·E | Auditor 级 TALE 门：对某一个档打分，适用时执行 T1/A1/L1/E1，产出 SHIP/FIX/BLOCK；full 模式保留三份彼此关联的结果。 |
| **Evaluate** | [message-test-designer](evaluate/message-test-designer/SKILL.md) | E | 设计信息测试 —— 变体矩阵、受众分组，以及战略叙事的共鸣读数。 |
| **Evaluate** | [narrative-resonance-monitor](evaluate/narrative-resonance-monitor/SKILL.md) | E | 用 keyless 数据源追踪叙事在各频道的落地情况（代理数据均标注）。 |
| **Evaluate** | [narrative-drift-monitor](evaluate/narrative-drift-monitor/SKILL.md) | E | 盯叙事漂移 —— 哪些频道已经偏离了已批准的正典 —— 并标记纠偏。 |

## 质量门 — TALE

[TALE](../references/tale-benchmark.md) 从四个维度评估品牌叙事 —— **T**ruth 真实性 · **A**rchitecture 结构 · **L**anding 落地 · **E**vidence 证据。它刻意**不**汇总成一个综合分：[`narrative-quality-auditor`](evaluate/narrative-quality-auditor/SKILL.md) 分别跑 **truth**、**system**、**effectiveness** 三个档，full 审查把三份结果关联起来但从不取平均。否决项 `T1`/`A1`/`L1`/`E1`（须带框架名限定 —— 这些 ID 与其他框架在文字上撞号）无论其余得分如何都会硬顶或阻断某个档。门的机制由全部八个 auditor 共享，见 [auditor-runbook.md](../references/auditor-runbook.md)。

## 快速开始

```text
/aaron-marketing:narrative              # 从你的输入推断合适的 TALE 阶段
/aaron-marketing:narrative --phase trace | architect | land | evaluate
```

```text
/aaron-marketing:narrative 我们在品类里总是听起来跟别人一样 —— 帮我们把故事改对
```

每个技能都在 **Tier 1** 用你粘贴的数据即可运行，无需任何 API key。

## 推荐场景

| 你的处境 | 从这里开始 | 得到什么 |
|---|---|---|
| "我们在品类里跟谁都一个样" | `/aaron-marketing:narrative --phase trace` | 品类叙事地图 + 你当前故事的诚实基线 |
| 高层重写了 pitch，各频道开始漂移 | `--phase evaluate` → `narrative-drift-monitor` | 每个频道偏离正典之处，附纠偏 |
| 融资或发布前需要一套统一信息体系 | `--phase architect` → `message-system-architect` | tagline、支柱、证据点、分受众角度，合成一个结构 |
| deck 里某条声明可能没有实证 | `positioning-truth-tracer` | 每条声明回溯到证据；无支撑的撤下（T1 否决） |
| 准备投资人/媒体叙事 | `--phase land` → `pitch-narrative-builder` | deck 主线、demo 故事、投资人/媒体框架 |
